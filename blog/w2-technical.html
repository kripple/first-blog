stuff I learned @ code review:

  • hashes are for labeled pieces of data
  • use hashes with descriptive keys
  • hashes are like arrays, just with more helpful indices (good for anyone that needs to read and understand your code)

when to use a data structure (string,hash,array,etc) vs. when to use a class:

if all you need to do is *hold state*, use a data struct
if you will ever need to *have behavior*, use a class

classes vs. modules:

use a module if you only need one instance, use classes if you need multiple

parsing examples:

def import(file) data = CSV.read(file,opts).to_a # CSV.read returns a CSV table thats in the format of an array of arrays & the first row is the headers. Need to convert to array to use array methods # opts: headers: true, header_converters: :symbol headers = data.shift # pops off headers row data.map do |person_data| Hash[headers.zip(person_data)] # crazy conversion magic end end def save(file, data) CSV.open(file,"w") do |csv| csv << data.first.keys data.each do |row| csv << row.values end end end

Here's another implementation:

module Parser def import(file) data = CSV.read(file).to_a # CSV.read returns a CSV table that's in the format of an array of arrays & the first row is the headers. Need to convert to array to use array methods data.map do |row| row end end def save(file, data) CSV.open(file,"w") do |csv| data.each do |row| csv << [row] # takes arrays only. CSV needs to know where the comma column separators go end end #File.write(file, new_file) end end

Then you do 'include Parser' in your class and 'require_relative 'parser'' at the top of your file
